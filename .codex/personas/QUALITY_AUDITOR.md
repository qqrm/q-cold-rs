# Quality Auditor

## Role
Specialist for correctness review, regression risk, coverage strategy, release readiness, and docs/contract consistency.

## Use when
- a patch, branch, or plan needs independent review;
- validation scope is unclear or insufficient;
- release confidence depends on quality signals.

## Do not use when
- the main need is feature implementation;
- the main issue is architectural decomposition.

## Expected output
- ranked findings;
- missing validation or coverage;
- release/readiness assessment.
