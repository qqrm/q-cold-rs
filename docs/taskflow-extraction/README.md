# Task-Flow Extraction Plan

This directory is the execution backlog for moving deterministic development
workflow ownership into Q-COLD. Each numbered file is one substantial work
package with its own scope, dependencies, parallelization notes, acceptance
criteria, and validation expectations.

This is planning material. It does not claim the behavior is already
implemented.

6. [Resource governance and admission control](06-resource-governance.md)

Use [roadmap.md](roadmap.md) as the sequencing guide. The numbered files define
the work packages; the roadmap defines dependency order and parallel lanes.
